# 一键导出公文 DOCX

> 本文档是 DOCX 导出功能的唯一规范来源。代码中的样式参数均以此为准。

## 做什么

点一下按钮，把当前 Markdown 文件变成一份符合 GB/T 9704-2012《党政机关公文格式》的 Word 文档。

不需要装 Pandoc，不需要跑大模型，一切都是确定性的机械转换。

## 怎么做

```
Markdown → remark 解析 → mdast 语法树 → ast-to-docx 映射 → docx 库打包 → .docx 文件
```

用 `docx` npm 包直接生成 OOXML，样式表只写你定义的那几条，不会冒出几十个用不着的英文样式。

代码分四层，各管各的事：

```
src/export/
├── constants.ts        页面尺寸、字号、行距、字体——全是命名常量
├── gbt9704-styles.ts   样式定义、页面布局、页脚
├── ast-to-docx.ts      语法树到 Word 文档的纯函数映射（不碰 VS Code API）
├── image-resolver.ts   图片路径解析、尺寸读取、版心缩放
└── export-command.ts   按钮点击后的交互流程编排
```

`ast-to-docx.ts` 是纯函数，拿到语法树就能出文档，跟 VS Code 没有半点关系，Jest 直接测。

## 版面规格

依据 GB/T 9704-2012。偏差允许 ± 1mm。

| 项目 | 数值 |
|------|------|
| 纸张 | A4 纵向（210 × 297mm） |
| 上边距 | 37mm |
| 下边距 | 35mm |
| 左边距 | 28mm |
| 右边距 | 26mm |
| 版心 | 156 × 225mm |
| 每页 | 22 行，每行 28 字 |
| 行距 | 固定值 28 磅（560 twip） |
| 正文首行缩进 | 2 个字符（640 twip） |
| 正文对齐 | 两端对齐 |

## 字体

公文里来来回回就这几种字体。每种中文字体配一个固定的英文搭档，代码里用拼音命名，一看就知道是什么。

| 变量名 | 中文 | 英文 | 用在哪 |
|--------|------|------|--------|
| `XiaoBiaoSong` | 方正小标宋简体 | Times New Roman | 公文大标题 |
| `HeiTi` | 黑体 | Arial | 一级标题、表头 |
| `KaiTi` | 楷体 | Arial | 二级标题、强调 |
| `FangSong` | 仿宋 | Times New Roman | 正文、三至七级标题 |
| `SongTi` | 宋体 | Times New Roman | 页码 |
| `CodeFont` | 宋体 | Consolas | 代码块 |

配对逻辑：黑体和楷体走无衬线路线，配 Arial；仿宋和宋体走衬线路线，配 Times New Roman。

方正小标宋系统不自带，字体文件放在 `fonts/FZXBSJW.TTF`。没装的话 Word 会自动回退到宋体。macOS 上 Word 会把 SimHei 回退到 Heiti SC、把 FangSong 回退到 STFangsong，不用在代码里操心平台差异。

所有字体一律不用 `_GB2312` 后缀。

## 标题

GB/T 9704 的标题层级靠字体区分，不靠字号大小。除了大标题用二号字，其余全是三号。

| 级别 | Markdown | Word 样式 | 字体 | 字号 | 加粗 | 对齐 |
|------|----------|-----------|------|------|------|------|
| 公文标题 | `#` | 标题 | 方正小标宋 | 二号 22pt | 否 | 居中，不缩进 |
| 一级 | `##` | 标题 1 | 黑体 | 三号 16pt | 否 | 左对齐，缩进 2 字符 |
| 二级 | `###` | 标题 2 | 楷体 | 三号 16pt | 否 | 同上 |
| 三级 | `####` | 标题 3 | 仿宋 | 三号 16pt | **是** | 同上 |
| 四级 | `#####` | 标题 4 | 仿宋 | 三号 16pt | 否 | 同上 |
| 五~七级 | `######` 等 | 标题 5-7 | 仿宋 | 三号 16pt | 否 | 同上 |

缩进 2 字符、回行顶格（GB/T 9704 第 7.3.3 条）。公文标题居中，不缩进。

各级标题的编号（"一、""（一）""1.""（1）"）写在 Markdown 文本里，不靠 Word 自动编号。

代码用 `styles.default.headingN` 覆盖 Word 内建样式。这一点至关重要——如果用 `paragraphStyles` 配自定义 `name`，Word 会把它当成新样式，导致样式面板里同时出现"标题 1"和你的自定义名称。

## 强调

Markdown 里的 `**加粗**` 导出后不是粗体，而是换成楷体。公文讲究用字体区分强调，不用粗细区分。

| 项目 | 值 |
|------|-----|
| 中文 | 楷体 |
| 英文 | Arial |
| 字号 | 三号 |
| 加粗 | 否 |

## 页码

GB/T 9704 第 7.5 条，逐字对照。

| 项目 | 规格 |
|------|------|
| 字体 | 宋体四号（14pt） |
| 数字 | 半角阿拉伯数字 |
| 一字线 | `—`（U+2014 Em Dash，占一个汉字宽度） |
| 格式 | —1— |
| 位置 | 版心下边缘之下，一字线上距版心 7mm |
| 奇数页 | 居右空一字 |
| 偶数页 | 居左空一字 |

页码里全是英文字符和 Em Dash，Word 实际渲染出来的是 Times New Roman。

## 表格

三线表风格。

| 项目 | 值 |
|------|-----|
| 宽度 | 撑满版心（100%） |
| 顶部、底部边框 | 1.5pt 实线 |
| 内部水平线 | 0.5pt 实线 |
| 垂直线 | 无 |
| 表头 | 黑体四号，居中 |
| 表体 | 仿宋四号 |

## 交互

编辑器右上角有个 Word 文档图标（dark/light 两套 SVG），只在 Markdown 文件里出现。

两种导出方式：

| 命令 | 入口 | 行为 |
|------|------|------|
| 导出公文 DOCX | 按钮 + 命令面板 | 弹出保存对话框 |
| 快速导出公文 DOCX | 命令面板 | 直接存到 .md 同目录 |

流程：检查未保存修改 → 保存对话框 → 进度条（可取消）→ 完成通知（含文件大小）→ "打开文件"或"在文件管理器中显示"。

导出路径自动记住，下次默认用上次的目录。出错时弹窗带"查看日志"按钮，点开就是 Output Channel 里的完整记录。

---

## 验收标准

- 导出的 DOCX 符合 GB/T 9704-2012 版面规格（纸张、边距、行距、字号）
- 标题层级字体映射正确（方正小标宋 / 黑体 / 楷体 / 仿宋）
- 表格为三线表风格，撑满版心
- 页码格式为 —N—，奇偶页位置正确
- 图片等比缩放不超出版心宽度
- 空文档、frontmatter、Mermaid 等边缘情况不崩溃

## 边缘情况

| 场景 | 处理 |
|------|------|
| 空文档 | 生成只有一个空段落的有效 DOCX |
| YAML Frontmatter | 跳过，不输出 |
| Mermaid 图表 | 等宽代码块 + 提示文字 |
| LaTeX 公式 | 等宽纯文本 |
| 远程图片 | 黑色占位文字 |
| 本地图片找不到 | 黑色占位文字 + 汇总警告 |
| 图片太宽 | 等比缩放到版心 156mm |
| 超链接 | 输出为黑色普通文字 |
| 文字颜色 | 统一黑色，绝无蓝色 |
